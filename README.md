## アプリケーションの作成方針

このアプリケーションは、就職活動を行うにあたり、これまでに学習してきたことを形にすることを目的として作成しました。
そのため、作成するアプリケーションを決定するにあたり、以下の 2 点を重視しました。

1. 自分が欲しいと思うものを作成する。
2. 個人の利用で完結するものを作成する。

#### 1. 自分が欲しいと思うものを作成する

本来、アプリケーションの存在意義は、ユーザーに対して価値を提供することであると思っています。

しかし、現段階の自身の企画力や知識量を鑑みるに、まだこの世にない新しいサービスを作り出すといったことや、多くの方のニーズに応えられるサービスを作成するといったことは現実的ではないと考えました。

また、あくまでもポートフォリオを作成する目的は、一定の技術力を有しており、自力で学習していく能力があるということを見ていただくことです。

この目的から逸脱しないためにも、既に類似するサービスが存在するか否か、需要があるか否かに関わらず、純粋に自分が欲しいと思う機能を詰め込んだアプリケーションを作成することにしました。
純粋に自分が欲しいものを作成することによって、要件の洗い出しに困ることなく、開発に着手することができました。

#### 2. 個人の利用で完結するものを作成する

初めは、SNS 系のアプリケーションを作成しようと考えていました。

しかし、誰かと繋がることを前提としたアプリケーションは、ユーザー数が相当数いなければ、サービスとして成り立ちません。
SNS 系のアプリケーションは、既に大手のものがありますし、それらを上回る魅力を提供するというのは簡単ではありませんから、過疎化した状態のアプリケーションになってしまうというのは想像に難くありません。

また、自分自身がそのようなアプリケーションを使いたいと思うのかというと、決してイエスとは言えませんでした。

アプリケーションを作成する以上は、少なくとも自分 1 人はユーザーとして使い続けたいという思いがあります。
自分が使いたいと思うものであれば、もしかしたら他の誰かも使いたいと思ってくれるかもしれません。

そのため、SNS 系のアプリケーションではなく、個人の利用で完結するアプリケーションを作成することにしました。

全体を通して、第一の目標はアプリケーションを使用可能な状態にすることでした。

そのため、選択肢が複数ある場合は、初心者にも分かりやすいこと、情報が豊富にあることを判断基準としました。

- バックエンドとフロントエンドを同一のビルドライフサイクルに乗せるために、[gradle-node-plugin](https://github.com/node-gradle/gradle-node-plugin) を使用して Gradle から npm コマンドが実行されるようにしました。

  gradle-node-plugin は Gradle のビルドの一部として、OS に応じた Node.js をプロジェクトローカルに自動ダウンロードし、任意の Node.js スクリプトを実行してくれるプラグインです。

  IntelliJ IDEA 上で spring-boot-devtools と併せて使用することで、フロントエンドとバックエンドのいずれを変更した場合も一度のキー入力で変更が反映されるため手早く確認でき、効率よく開発を進めることができました。

  また、Docker や AWS の知識が浅く、戸惑うことが多かったので、フロントエンドとバックエンドが一つの jar ファイルに同梱されることで、環境構築がシンプルになったという点でもよかったなと思います。

  ただ、パフォーマンス面で問題があるようであれば変更していこうと考えています。

#### 自動デプロイの内訳

- GitHub の develop ブランチにプッシュした場合

  (1). アプリケーションのテストを実行する

- main ブランチにマージした場合

  (1). アプリケーションをビルドして、実行可能 jar を生成する

  (2). 生成された実行可能 jar を含む Docker イメージをビルドする

  (3). 生成された Docker イメージを AWS ECR へプッシュする

  (4). AWS ECS のサービスを更新する(本番環境にデプロイ)

# 最後に

実際にアプリを作成してみて痛感したのは、「作りたいものを思った通りに作り上げる難しさ」です。
当然な話ですが、「こんな機能が欲しい」と思っても、技術が無ければ実現できません。
独学でアプリを作成していく中で、何度も壁にぶち当たりました。正直、実現できていないものもあります。その困難さに直面して、逆にモチベーションが非常に上がりました。まだまだ未熟ですが、今後も努力を継続していきます。

高卒未経験でも仕事を続けながら学習を続けられた要因として、
プログラムを書いたり、どうやったら実装できるのかを考えている時間が、楽しいと感じたからだと思います。
今回作成した PF はまだまだ改良中で、無理やり実装している部分や妥協している点も多いので、学習を続けて知識を深めながら、転職活動に望みたいと思います。
又この記事について、間違っている箇所があればご指摘頂けると幸いです。
